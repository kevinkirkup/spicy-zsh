# ZSH Configuration Reorganization — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Reorganize the ZSH dotfiles repo from a `pre/`/`setup/`/`post/` directory ordering scheme into 5 named core files + a `tools/` directory, with platform differences collapsed into inline guards.

**Architecture:** The new `.zshrc` is an explicit source manifest. Five thematic core files (`omz.sh`, `path.sh`, `exports.sh`, `aliases.sh`, `functions.sh`) consolidate the `pre/` and `post/` platform trees. All tool-specific configs migrate to `tools/` (one file per tool, platform forks folded inline). New files are created first; `.zshrc` is swapped last so the live shell is never broken mid-migration.

**Tech Stack:** ZSH, oh-my-zsh, homesick (symlink manager). All files under `home/` map to `~/` via `homesick link spicy-zsh`.

  if [ ! -z "$1" ]; then
    history | grep "$1" | grep -v histg

function json2yaml() {
  local JSON_FILE=$1
  local YAML_FILE="${JSON_FILE%.json}.yml"
  python -c 'import sys, yaml, json; yaml.safe_dump(json.load(sys.stdin), sys.stdout, default_flow_style=False)' < ${JSON_FILE} > ${YAML_FILE}
}

function yaml2json() {
  local YAML_FILE=$1
  local JSON_FILE="${YAML_FILE%.*}.json"
  python -c 'import sys, yaml, json; json.dump(yaml.safe_load(sys.stdin), sys.stdout, indent=4)' < ${YAML_FILE} > ${JSON_FILE}

  local BUILD_DATE=$(date -u +%FT%T)
  echo $BUILD_DATE | sed 's/[T:-]//g'

function extract_arch_framework() {
  local FRAMEWORK_EXECUTABLE_PATH=$1
  local ARCHS=$2
  local EXTRACTED_ARCHS=()
  for ARCH in $ARCHS; do
    lipo -extract "$ARCH" "$FRAMEWORK_EXECUTABLE_PATH" -o "$FRAMEWORK_EXECUTABLE_PATH-$ARCH"
    EXTRACTED_ARCHS+=("$FRAMEWORK_EXECUTABLE_PATH-$ARCH")
  done
  lipo -o "$FRAMEWORK_EXECUTABLE_PATH-merged" -create "${EXTRACTED_ARCHS[@]}"
  rm "${EXTRACTED_ARCHS[@]}"
}
